# 安全审查清单 — OWASP 对齐

进行安全专项审查或安全维度需要更深入分析时阅读此参考文件。按 OWASP Top 10（2025）分类组织。

---

## A01：访问控制失效

- [ ] 每个 API 端点都有授权检查（不仅仅是前端）
- [ ] IDOR 防护：用户只能访问自己的资源
- [ ] CORS 配置：基于白名单，非通配符
- [ ] 路径遍历防护：路径规范化
- [ ] JWT 验证：签名、过期时间、签发者、受众
- [ ] 敏感端点的速率限制（登录、密码重置）
- [ ] 服务账户应用最小权限原则
- [ ] 管理功能通过角色检查保护，而非 URL 隐蔽性

## A02：加密失败

- [ ] 所有通信强制使用 TLS 1.2+
- [ ] 仅使用强算法：AES-256-GCM、ChaCha20-Poly1305
- [ ] 禁止已弃用的：DES、3DES、RC4、用于安全目的的 MD5
- [ ] 密钥管理：密钥不在源代码中，定期轮换
- [ ] 密码哈希：bcrypt/Argon2/PBKDF2 配合唯一盐值
- [ ] 随机数生成：加密安全 PRNG（非 Math.random()）
- [ ] 证书验证：生产环境中不禁用验证
- [ ] 数据库中 PII 静态加密

## A03：注入

- [ ] SQL：所有地方使用参数化查询（无字符串拼接）
- [ ] NoSQL：查询对象净化
- [ ] XSS：针对特定上下文的输出编码（HTML/JS/URL/CSS）
- [ ] 命令注入：避免 shell 执行；如必需，严格白名单
- [ ] 路径遍历：规范化路径，限制在允许的目录内
- [ ] LDAP 注入：正确转义过滤器输入
- [ ] 模板注入：沙箱化模板引擎，禁止用户控制的模板
- [ ] 头注入：在 HTTP 头之前验证/去除用户输入中的 CRLF

## A04：不安全设计

- [ ] 新功能进行了威胁建模
- [ ] 考虑了业务逻辑滥用场景（优惠券重用、双花）
- [ ] 组件之间清晰定义了信任边界
- [ ] 安全失败的默认值（出错时拒绝访问，而非授予）
- [ ] 防止滥用的资源限制（文件上传大小、API 速率限制）

## A05：安全配置错误

- [ ] 默认凭证已移除/更改
- [ ] 生产环境禁用调试模式
- [ ] 错误信息不暴露堆栈跟踪或内部细节
- [ ] 安全头已设置（CSP、HSTS、X-Frame-Options 等）
- [ ] 不必要的功能/端口/服务已禁用
- [ ] 云 IAM 权限遵循最小权限
- [ ] 日志配置不包含敏感数据

## A06：存在漏洞和过时的组件

- [ ] 依赖锁文件存在并已提交
- [ ] 直接或传递依赖中无已知 CVE
- [ ] CI 中有自动化漏洞扫描（npm audit、pip-audit 等）
- [ ] 许可证兼容性已验证
- [ ] 已识别需要迁移的生命周期终止框架/库

## A07：身份识别和认证失败

- [ ] 敏感账户可使用多因素认证
- [ ] 登录后重新生成会话 Token
- [ ] 配置了会话超时和空闲超时
- [ ] 服务端强制执行密码复杂度要求
- [ ] 重复失败后锁定账户（配合速率限制）
- [ ] 凭证填充防护（验证码、设备指纹）
- [ ] "记住我" Token 安全保护（与会话分离）

## A08：软件和数据完整性失败

- [ ] CI/CD 流水线已保护（签名提交、受保护分支）
- [ ] 依赖完整性已验证（锁文件哈希、子资源完整性）
- [ ] 避免或沙箱化不受信任数据的反序列化
- [ ] 自动更新机制验证签名
- [ ] 分发制品的代码签名

## A09：安全日志和监控失败

- [ ] 认证事件已记录（成功、失败、锁定）
- [ ] 授权失败已记录
- [ ] 输入验证失败已记录
- [ ] 高价值交易已记录并附上下文
- [ ] 防止日志注入（日志中用户输入已净化）
- [ ] 日志安全存储，防篡改
- [ ] 配置了可疑模式的告警

## A10：服务端请求伪造（SSRF）

- [ ] 用户提供的 URL 通过白名单验证
- [ ] 内部网络范围已屏蔽（RFC 1918、链路本地、环回）
- [ ] URL 协议限制（仅 http/https）
- [ ] DNS 重绑定防护
- [ ] 云元数据端点已屏蔽（169.254.169.254）

## 各语言特定安全陷阱

### JavaScript / TypeScript
- 通过 `__proto__` 或 `constructor.prototype` 的原型污染
- 使用用户输入的 `eval()`、`Function()` 和 `new Function()`
- ReDoS（正则表达式拒绝服务）
- 未验证来源的 `window.postMessage`
- 未经净化的 `dangerouslySetInnerHTML`（React）

### Python
- 对不受信任数据使用 `pickle.loads()`（任意代码执行）
- 未使用 `Loader=SafeLoader` 的 `yaml.load()`
- 通过 `.format()` 或带用户数据的 f-string 的格式字符串注入
- Django raw queries 或 SQLAlchemy text() 的 SQL 注入
- 使用用户输入的 `subprocess.shell=True`

### Java / Kotlin
- 不受信任 ObjectInputStream 的反序列化
- 默认 XML 解析器的 XML 外部实体（XXE）
- JPA 字符串拼接的 SQL 注入
- Spring Security 配置错误（默认允许）
- `java.util.Random` 的不安全随机（应使用 `SecureRandom`）

### Go
- 查询中通过 `fmt.Sprintf` 的 SQL 注入（应使用带参数的 `db.Query`）
- `filepath.Join` 未配合 `filepath.Clean` 的路径遍历
- 整数溢出（Go 不会在溢出时 panic）
- 无限制生成导致的 goroutine 泄漏
- 不当的 TLS 配置（InsecureSkipVerify）

### Rust
- `unsafe` 块：每个都需要理由和审计
- 生产代码中的 `.unwrap()`（应使用正确的错误处理）
- release 模式下的整数溢出（静默回绕）
- unsafe 块中的原始指针解引用
- FFI 边界：C/Rust 接口处的数据验证

### PHP
- 比较中的类型转换陷阱（`==` vs `===`）
- 用户输入上的 `unserialize()`（对象注入）
- 用户控制路径的文件包含（`include`、`require`）
- 用户输入上的 `extract()`（变量注入）
- 非预处理语句的 SQL 注入
